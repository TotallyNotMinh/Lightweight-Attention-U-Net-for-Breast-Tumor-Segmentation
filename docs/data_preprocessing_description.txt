This document provides a highly detailed, comprehensive breakdown of the data
loading, preprocessing, and augmentation pipeline implemented in the project.
Medical ultrasound segmentation datasets are typically small, highly variable
in contrast, and contain significant acoustic noise and artifacts. The pipeline
analyzed below represents a robust solution tailored to these challenges.

--------------------------------------------------------------------------------
1. DATASET INTEGRATION AND DIRECTORY STRUCTURE
--------------------------------------------------------------------------------
The dataset class `BreastUltrasoundDataset` initializes automatically by:
  A. Automated Download: Leveraging `kagglehub.dataset_download` to fetch the
     public Breast Ultrasound Images Dataset ("aryashah2k/breast-ultrasound-images-dataset").
  B. Class Subdirectories: Locating files in "Dataset_BUSI_with_GT/benign"
     and "Dataset_BUSI_with_GT/malignant".
  C. Multi-Mask Matching:
     * First, it collects all primary image files (ending in `.png` and NOT containing
       the substring `_mask`).
     * For each image, it scans the directory for all associated ground-truth mask files
       whose names match the format: `base_name_mask*.png` (e.g. `benign (1).png` matches
       `benign (1)_mask.png`, `benign (1)_mask_1.png`, etc.). This is a critical design
       feature, as some patients present with multiple distinct lesions represented
       by separate mask files.
     * Samples are aggregated into a list of dictionaries:
       `{"image": img_path, "masks": mask_paths, "label": cls}`

--------------------------------------------------------------------------------
2. DETAILED PREPROCESSING WORKFLOW (PER SAMPLE)
--------------------------------------------------------------------------------
When a dataset item is fetched (`__getitem__`), the following steps are executed:

A. Image Loading (Grayscale):
   The primary ultrasound image is loaded as a 2D grayscale array using OpenCV:
   `cv2.imread(sample["image"], cv2.IMREAD_GRAYSCALE)` -> shape: (H, W).
   Loading in grayscale avoids redundant channel processing and matches the 1-channel
   input configuration of the UNet.

B. Multi-Mask Combination (`combine_masks`):
   To represent all lesions in a single ground-truth tensor:
   - A blank canvas of shape (H, W) is initialized with zeros (`np.uint8`).
   - For each associated mask file, it is loaded in grayscale (`cv2.IMREAD_GRAYSCALE`).
   - The mask is binarized by checking values > 0 (`(mask > 0).astype(np.uint8)`).
   - Masks are combined using a pixel-wise maximum:
     `combined_mask = np.maximum(combined_mask, mask)`
   - This merges all disjoint lesion regions into a single binarized mask tensor.

C. Symmetric Aspect-Ratio Padding (`pad_to_square`):
   Ultrasound scans are captured with various rectangular resolutions. Directly
   resizing them would squish or stretch the anatomical structures, distorting lesion
   shapes and speckle patterns.
   - The method computes `size = max(h, w)`.
   - The height/width difference is divided symmetrically: `pad_h = size - h`, `pad_w = size - w`.
   - Black pixels (constant border value = 0) are added on the borders:
     * top = pad_h // 2, bottom = pad_h - top
     * left = pad_w // 2, right = pad_w - left
   - This guarantees a perfect square shape while maintaining the original aspect ratio.

D. Resizing:
   - Image: Resized to (image_size, image_size) (default 256x256) using **Bilinear Interpolation**
     (`cv2.INTER_LINEAR`).
   - Mask: Resized to (image_size, image_size) using **Nearest Neighbor Interpolation**
     (`cv2.INTER_NEAREST`). This is critical: nearest-neighbor interpolation prevents
     creating intermediate fractional values (e.g. 0.5) at mask boundaries, keeping the
     resulting mask strictly binary.

E. CLAHE Preprocessing (Contrast Enhancement):
   Ultrasound images suffer from low contrast, acoustic shadow zones, and inconsistent
   gain.
   - The class instantiates CLAHE via `cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8, 8))`.
   - CLAHE divides the image into a grid of 8x8 local tiles, computes histograms,
     and equalizes them locally. The contrast limit of 2.0 prevents noise amplification
     in homogeneous background regions.
   - This boosts the visibility of lesion margins and normalizes image brightness variations.

F. Normalization and Tensor Preparation:
   - The enhanced image is cast to float32 and normalized: `image.astype(np.float32) / 255.0`
     (maps pixel intensities from [0, 255] to [0.0, 1.0]).
   - The combined mask is thresholded: `(mask > 0).astype(np.float32)`.
   - Dimension expansion: Channel dimensions are explicitly added:
     * Mask shape: (1, H, W)
     * Image shape: (1, H, W)
   - Outputs are returned as PyTorch float tensors.

3. THE MEDICAL-SPECIFIC DATA AUGMENTATION PIPELINE
--------------------------------------------------------------------------------
Data augmentation is applied inside the `augment` function (when training is enabled).
It uses a highly advanced set of randomized operations tailored for medical imaging:

- Horizontal Flip (30% probability):
  * Command: `cv2.flip(image, 1)`, `cv2.flip(mask, 1)`
  * Simulates variation in lateral transducer placement.

- Vertical Flip (30% probability):
  * Command: `cv2.flip(image, 0)`, `cv2.flip(mask, 0)`
  * Simulates inverted scans or tissue layering directions.

- Random Rotation (30% probability):
  * Random angle in [-30, 30] degrees.
  * Extrapolates rotation matrix around image center: `cv2.getRotationMatrix2D((w//2, h//2), angle, 1)`.
  * Warp applied: Bilinear for image (`cv2.INTER_LINEAR`), Nearest for mask (`cv2.INTER_NEAREST`).
  * Simulates varying transducer angles during clinical scans.

- Intensity Contrast & Brightness Jitter (30% probability):
  * Scales contrast with multiplier `alpha` in range [0.7, 1.3].
  * Shifts brightness with offset `beta` in range [-20, 20] (integer values).
  * Applied via `cv2.convertScaleAbs`, which automatically clamps and clips results between 0 and 255.
  * Simulates machine settings differences (gain adjustments) and varying tissue attenuation.

- Gaussian Speckle/Thermal Noise (30% probability):
  * Random standard deviation `sigma` is sampled from [2.0, 25.0].
  * Noise is generated via `np.random.normal(0, sigma, shape)`.
  * Added to raw image, clipped, and cast back to uint8.
  * Simulates high acoustic speckle and electronic thermal noise inherent to ultrasound.

- Gaussian Blur (30% probability):
  * Randomly blurs image using kernel size 3x3 or 5x5 (`cv2.GaussianBlur`).
  * Simulates differing transducer focus depths or out-of-focus acquisition.

- Single-Pass Scale Jitter (30% probability):
  * Samples zoom scale from [0.8, 1.2].
  * Resizes image and mask to the scaled dimensions.
  * If scale > 1.0 (zoom-in): Crops the center back to original size.
  * If scale < 1.0 (zoom-out): Symmetrically pads with zeros back to original size.
  * Avoids applying double interpolation passes, which prevents pixel blurring.
  * Simulates varying patient-to-transducer distance and biological size diversity.

- Elastic Deformation (50% probability) [CRITICAL FOR MEDICAL DATA]:
  * Simulates actual physical tissue compression under the ultrasound probe.
  * Generates random displacement vectors dx, dy of shape (H, W).
  * Smooths the fields using a Gaussian filter with standard deviation `sigma` in [6, 10]
    and multiplies by displacement scale `alpha` in [30, 60].
  * Warps the pixels using `cv2.remap` (bilinear for image, nearest for mask).
  * Introduces localized, organic anatomical warpings without destroying boundaries.

- Coarse Dropout / Cutout (50% probability):
  * Randomly select between 1 and 4 rectangular holes.
  * Hole sizes are randomized between 16 and 40 pixels.
  * Fills selected patches in the image with black (0).
  * Simulates acoustic shadowing (commonly occurring behind highly attenuating calcified structures)
    and physical signal dropouts from poor gel contact.

--------------------------------------------------------------------------------
4. COMPARISON: ROOT load_data.py vs. DATA/load_data.py
--------------------------------------------------------------------------------
There are two versions of `load_data.py` present in the project. The version
located at `data/load_data.py` is the version imported by the training notebook
(`train.ipynb`), representing a major evolution over the root file.

Key differences:

A. Image Channel Count:
   - Root: Loads images in color (`cv2.IMREAD_COLOR`), and outputs shape (3, H, W).
     This introduces a structural mismatch since the UNet encoder is instantiated with
     `in_chans=1`.
   - data/load_data.py: Loads images strictly in grayscale (`cv2.IMREAD_GRAYSCALE`),
     outputting (1, H, W). This perfectly matches the UNet architecture.

B. Directory Assumption:
   - Root: Expects external inputs `image_dir` and `mask_dir` and looks specifically for
     `bus_xxxxx.png` -> `mask_xxxxx.png`.
   - data/load_data.py: Automates the dataset download and directory management internally,
     and supports class-based grouping (benign, malignant).

C. Mask Aggregation:
   - Root: Only reads a single file matching `mask_xxxxx.png`.
   - data/load_data.py: Scans and merges multiple mask files matching `base_name_mask*.png`
     to support patients with multiple lesions.

D. CLAHE Contrast Enhancement:
   - Root: Lacks any native contrast enhancement.
   - data/load_data.py: Employs CLAHE, significantly improving raw feature readability.

E. Augmentation Diversity:
   - Root: Employs only standard basic augmentations (horizontal flip, vertical flip,
     rotation +/-15 deg, simple intensity scaling [0.8, 1.2], simple Gaussian noise with
     standard deviation 0.01).
   - data/load_data.py: Employs an extremely comprehensive medical-specific augmentation
     suite, including Gaussian Blur, Single-Pass Scale Jitter, Coarse Dropout (Cutout)
     to simulate acoustic shadows, and Elastic Deformations to simulate tissue compression
     under transducer pressure.

Conclusion:
The pipeline in `data/load_data.py` is significantly more advanced and represents a
production-grade preprocessing pipeline optimized for deep learning on ultrasound scans.
